- 交付文件:`fund-valuation.html`(单文件纯前端应用)。
- 需求:按代码添加基金到分组,基于每季度最新公布持仓比例做实时估值。
- 技术方案:全部用公开接口 + script 注入(JSONP 思路)绕过跨域,**无需注册任何 API**。

## bat 编码报错修复(第五轮)
- 现象:双击 bat 报 `'㈡祴' 不是内部或外部命令`、`此时不应有 &` 等乱码。
- 根因:① bat 文件存成 UTF-8,但中文版 Windows cmd 按 GBK(代码页936)解析,中文字节被错切成命令;② 文件用 LF 换行,导致部分行粘连(如 `cd /d "%~dp0"echo.`);③ `start "" /b cmd /c "... & start ..."` 的 & 嵌套在 cmd 解析有陷阱。
- 修复:bat 重写为纯 GBK(GB2312)编码 + CRLF 换行,去掉 `chcp 65001`(GBK 文件在中文cmd天然正常);`&` 嵌套改为 `start "" cmd /c "timeout /t 2 >nul & start "" http://localhost:8765/"`。
- 关键教训:**在中文 Windows 上写 .bat 含中文,必须用 GBK + CRLF,绝不用 UTF-8/LF。** 用 PowerShell `[System.Text.Encoding]::GetEncoding("GB2312")` + `[System.IO.File]::WriteAllBytes` 写入。
- 实测:Start-Process 跑 bat,4秒后 Invoke-WebRequest 拿到 HTTP 200,内容正常。

## 第六轮:缓存导致"添加没反应" + bat中文乱码
- 现象:用户双击 bat 成功(端口占用提示=server.js正常跑),但页面输入代码点添加无反应;bat 中文显示 `?????`。
- 根因1(添加无反应):浏览器缓存了旧版 HTML(带死锁bug的版本),没拿到修复后的新版。解决:server.js 加 no-cache 响应头(Cache-Control: no-store/no-cache,Pragma,Expires:0),HTML 加 v2.2 版本标记便于确认。
- 根因2(中文乱码):用户cmd代码页非936,GBK文件中文显示成?。解决:bat 开头加 `chcp 936 >nul` 强制中文代码页。
- 验证:HTTP 200,内容含 v2.2,Cache-Control 头生效;bat GBK 解码中文完全正常。
- 注意:每次我后台为预览起的 server.js 会占用 8765,用户双击 bat 会撞端口。交付前需 Stop-Process 释放端口,或告诉用户"已占用可直接打开浏览器"。

## 第八轮:找到并修复真凶(TDZ)★★★
- 现象升级:添加基金"完全没反应,连dbg都不出"。
- 用 Chrome headless(`chrome --headless=new --enable-logging=stderr --v=1 --dump-dom URL`)抓到 Console:**`Uncaught ReferenceError: Cannot access 'storageOK' before initialization`**。
- 真凶:`let state = load();` 在顶层立即执行,load() 内首行用 `storageOK=checkStorage()`,但 `let storageOK=true` 声明在其后 → **let 的暂时性死区(TDZ)** 抛错,整个脚本从该行崩溃,后续所有函数/绑定全没执行,故按钮点击无任何反应。
- 修复:把所有 `let 变量声明`(storageOK/fundCache/stockCache/expanded)移到 `let state=load()` 和 `let series=loadSeries()` 之前。版本 v2.5。
- 顺带重构:全局 error 捕获显示到页面;所有事件绑定统一进 init()(DOMContentLoaded 触发)+ 每个绑定用 bind() 带存在性检查;导出/导入/新建组改为命名函数 doExport/doImportFile/doNewGroup。
- 验证方法(复用):给页面加 `?selftest=1` 钩子自动执行 addFund 并把结果写 document.title(SELFTEST:PASS:before=0:after=1),headless dump-dom 读 title。实测 PASS。
- 教训:**顶层 `let a = fnUsingVarB()` 时,varB 的 let 声明必须在前**,否则 TDZ 崩整个脚本且无声。排查这类"整个JS不执行"必用 headless 抓 console,不要靠肉眼读代码。
